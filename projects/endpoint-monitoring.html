PROJECT

Endpoint Monitoring & Detection Lab

SIEM / WAZUH HOME LAB

Overview

This project covers hands-on design and deployment of an endpoint monitoring and detection lab using a Security Information and Event Management (SIEM) platform. The lab provides real-time monitoring and threat detection across multiple endpoints to catch malicious activity and maintain a secure environment.

Technologies Used

  • Wazuh for event collection and threat detection
  • VirtualBox for virtualization and network isolation
  • Windows and Ubuntu as endpoints and manager
  • File Integrity Monitoring (FIM) for detecting unauthorized file changes

Key Features

  • Real-time event correlation and alerting
  • File integrity monitoring for system protection
  • Network traffic monitoring and analysis
  • Custom security rules to detect endpoint anomalies

Build Notes

I set up a virtualized environment with Wazuh agents on Windows endpoints, forwarding system-level security events to a centralized manager running on Ubuntu. Work centered on implementing file integrity monitoring and writing custom alerting rules for suspicious activity, alongside network traffic monitoring and log analysis for indicators of compromise. The build also involved troubleshooting network configuration and agent communication to keep the system reliable.

Challenges & Solutions

Issue: Configuring agent communication in a virtualized environment that required network isolation.

Solution: Implemented host-only networking in VirtualBox so the VM could communicate securely while staying isolated.

Future Improvements

  • Add advanced threat detection for more sophisticated attack patterns
  • Build a centralized dashboard for endpoint status visualization

Setup Steps

  • Clone the lab repository
  • Install Wazuh on both the server and client VMs
  • Configure the agent and server per the repository config files
  • (Optional) Run network traffic analysis with Wireshark
