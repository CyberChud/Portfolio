Project 2: Endpoint Monitoring & Detection Lab

Overview

This project highlights my hands-on experience in designing and deploying an Endpoint Monitoring and Detection Lab using a Security Information and Event Management (SIEM) platform. The lab focuses on ensuring real-time monitoring and threat detection across multiple endpoints to detect malicious activity and maintain a secure environment.

Technologies Used

  • Wazuh for event collection and threat detection
  • VirtualBox for virtualization and network isolation
  • Windows and Ubuntu as endpoints and manager
  • File Integrity Monitoring (FIM) for detecting unauthorized file changes

Key Features

  • Real-time event correlation and alerting
  • File Integrity Monitoring (FIM) for system protection
  • Network Traffic Monitoring for capturing and analyzing traffic
  • Custom security rules to detect endpoint anomalies

Project Description

In this project, I set up a virtualized environment with Wazuh agents on Windows endpoints. The agents monitor system-level security events and send logs to a centralized manager running on Ubuntu. I focused on implementing file integrity monitoring (FIM) and designed custom alerting rules to detect suspicious activities. Additionally, the setup included monitoring network traffic and analyzing event logs for indicators of compromise (IOCs). The lab also involved troubleshooting network configurations and agent communication to ensure the system's reliability.

Challenges & Solutions

Issue: Configuring agent communication in a virtualized environment where network isolation was required.

Solution: I implemented host-only networking in VirtualBox to allow the VM to communicate securely while maintaining isolation.

Future Improvements

  • Add advanced threat detection features to monitor for sophisticated attacks.
  • Implement centralized dashboarding for better visualization of endpoint statuses.

How to Set Up the Lab

  1. Clone this repository.
  2. Install Wazuh on both the server and client VMs.
  3. Configure the agent and server as per the configuration files in the repository.
  4. (Optional) Run network traffic analysis using Wireshark.
