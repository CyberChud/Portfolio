# New Project Questionnaire

Fill this out for any new project before adding it to the site. Answers map directly onto the case-study section structure used in `projects/micro-pentest.html` and `projects/endpoint-monitoring.html` — copy that file's HTML structure, then fill each section from the answers below. Skip a section (or mark it `[CONTENT NEEDED]`) rather than guessing.

1. **What problem did this solve?**
2. **Was this professional, academic, or personal?** (This decides whether it belongs under Security Engineering / Security Automation / Web & Technical Projects on the Projects page, and whether it's labeled "personal lab" anywhere.)
3. **What did I personally implement?** (vs. what was pre-built, a team effort, or a template)
4. **What technologies were used?**
5. **What security controls were added, if any?**
6. **How was it tested?** (specific test cases/events, not just "it worked")
7. **What failed, or what had to be reworked?**
8. **What tradeoffs were made, and why?**
9. **What measurable outcome occurred?** (only include numbers you can stand behind if asked about them in an interview)
10. **What evidence can be safely published?** (screenshots, diagrams, repo links — check for confidential info, client names, IPs, credentials first)
11. **What would I improve next?**
12. **Status:** Completed / Ongoing / Planned / In Development

## Case-study sections (fill from the answers above)

Overview · Problem · Objective · Environment · Architecture · My Responsibilities · Technical Implementation · Security Controls · Testing & Validation · Challenges · Decisions & Tradeoffs · Results · Lessons Learned · Future Improvements · Screenshots & Diagrams · Technologies · Repository / Demo

Use `[CONTENT NEEDED]`, `[ADD METRIC]`, `[ADD SCREENSHOT]`, or `[ADD DIAGRAM]` for anything not yet answered — don't leave a section silently blank, and don't fill it with invented specifics.
